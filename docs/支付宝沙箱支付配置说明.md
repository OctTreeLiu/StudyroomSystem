# 支付宝沙箱支付配置说明

## 一、获取支付宝沙箱账号信息

1. 访问支付宝开放平台：https://open.alipay.com/
2. 登录后进入"控制台" -> "开发助手" -> "沙箱环境"
3. 在沙箱环境中，你可以看到：
   - **APPID**：应用ID
   - **应用私钥**：需要自己生成
   - **支付宝公钥**：沙箱环境提供的公钥

## 二、生成应用私钥

### 方法一：使用支付宝提供的工具（推荐）

1. 下载支付宝提供的密钥生成工具：https://opendocs.alipay.com/common/02kkv7
2. 运行工具，选择"RSA2(SHA256)密钥"，点击"生成密钥"
3. 保存生成的**应用私钥**（用于配置）
4. 将生成的**应用公钥**复制到支付宝开放平台的"沙箱应用" -> "接口加签方式" -> "设置" -> 粘贴公钥

### 方法二：使用OpenSSL命令行

```bash
# 生成私钥
openssl genrsa -out app_private_key.pem 2048

# 生成公钥
openssl rsa -in app_private_key.pem -pubout -out app_public_key.pem

# 查看私钥内容（需要配置到application.yml）
cat app_private_key.pem
```

**注意**：私钥格式需要是PKCS8格式，如果生成的是PKCS1格式，需要转换：

```bash
# 转换为PKCS8格式
openssl pkcs8 -topk8 -inform PEM -in app_private_key.pem -outform PEM -nocrypt -out app_private_key_pkcs8.pem
```

## 三、配置后端

### 1. 修改 `backend/src/main/resources/application.yml`

```yaml
# 支付宝配置
alipay:
  app-id: 你的APPID  # 从支付宝开放平台获取
  app-private-key: |
    -----BEGIN PRIVATE KEY-----
    你的应用私钥内容（多行）
    -----END PRIVATE KEY-----
  alipay-public-key: |
    -----BEGIN PUBLIC KEY-----
    支付宝公钥内容（从沙箱环境获取）
    -----END PUBLIC KEY-----
  sign-type: RSA2
  charset: UTF-8
  gateway-url: https://openapi.alipaydev.com/gateway.do  # 沙箱环境地址
  return-url: http://localhost:3000/user/reservation?paySuccess=true  # 支付成功后跳转地址
  notify-url: http://你的服务器IP:8080/api/reservation/pay/notify  # 支付异步通知地址
```

### 2. 私钥配置格式说明

**重要**：私钥和公钥需要保持完整的格式，包括：
- `-----BEGIN PRIVATE KEY-----`
- 密钥内容（多行）
- `-----END PRIVATE KEY-----`

示例：
```yaml
app-private-key: |
  -----BEGIN PRIVATE KEY-----
  MIIEvQIBADANBgkqhkiG9w0BAQEFAASCBKcwggSjAgEAAoIBAQC...
  ...（多行内容）...
  -----END PRIVATE KEY-----
```

### 3. 配置回调地址

#### 本地开发环境

- **return-url**：前端地址，支付成功后用户跳转的页面
- **notify-url**：后端地址，支付宝服务器异步通知的地址

如果使用本地开发：
- 可以使用内网穿透工具（如ngrok、natapp）将本地服务暴露到公网
- 或者使用支付宝提供的沙箱测试工具进行测试

#### 生产环境

- **return-url**：`https://你的域名/user/reservation?paySuccess=true`
- **notify-url**：`https://你的域名/api/reservation/pay/notify`

## 四、测试支付流程

### 1. 启动后端服务

```bash
cd backend
mvn spring-boot:run
```

### 2. 启动前端服务

```bash
cd frontend
npm run dev
```

### 3. 使用沙箱账号测试

1. 在支付宝沙箱环境中，有提供的**买家账号**和**卖家账号**
2. 使用买家账号登录支付宝沙箱版APP进行支付测试
3. 沙箱账号信息在"沙箱环境" -> "沙箱账号"中查看

### 4. 测试步骤

1. 创建预约订单
2. 点击"支付"按钮
3. 会跳转到支付宝支付页面（沙箱环境）
4. 使用沙箱买家账号登录并支付
5. 支付成功后会自动跳转回系统

## 五、常见问题

### 1. 签名验证失败

**原因**：
- 私钥格式不正确
- 公钥配置错误
- 密钥不匹配

**解决方法**：
- 检查私钥是否为PKCS8格式
- 确认应用公钥已正确配置到支付宝开放平台
- 确认支付宝公钥是从沙箱环境正确复制

### 2. 回调地址无法访问

**原因**：
- 本地开发环境无法接收支付宝服务器的回调

**解决方法**：
- 使用内网穿透工具（ngrok、natapp等）
- 或者先测试同步回调（return-url），异步回调（notify-url）需要公网地址

### 3. 支付表单无法显示

**原因**：
- 网关地址配置错误
- APPID配置错误

**解决方法**：
- 确认 `gateway-url` 为 `https://openapi.alipaydev.com/gateway.do`（沙箱环境）
- 确认 `app-id` 正确

### 4. 支付成功但订单状态未更新

**原因**：
- 异步回调未正确处理
- 回调地址无法访问

**解决方法**：
- 检查后端日志，查看是否有回调请求
- 确认 `notify-url` 配置正确且可访问
- 检查回调处理逻辑

## 六、生产环境配置

### 1. 切换到正式环境

修改 `application.yml`：

```yaml
alipay:
  gateway-url: https://openapi.alipay.com/gateway.do  # 正式环境地址
  app-id: 你的正式APPID
  # 其他配置保持不变，但需要使用正式环境的密钥
```

### 2. 申请正式应用

1. 在支付宝开放平台申请正式应用
2. 提交审核
3. 审核通过后获取正式环境的APPID和密钥

### 3. 配置正式环境密钥

- 在正式应用中配置应用公钥
- 获取支付宝公钥（正式环境）
- 更新配置文件

## 七、安全建议

1. **不要将私钥提交到代码仓库**
   - 使用环境变量或配置文件（不提交到Git）
   - 或使用配置中心管理敏感信息

2. **使用HTTPS**
   - 生产环境必须使用HTTPS
   - 确保回调地址使用HTTPS

3. **验证回调签名**
   - 当前实现已包含签名验证
   - 确保回调处理逻辑正确

4. **订单状态幂等性**
   - 确保同一订单不会重复处理
   - 检查订单状态后再更新
