# 逐光自习室管理系统 — 部署说明

本文档面向**首次部署**的场景，按「功能介绍 → 环境要求 → 部署步骤 → 常见问题」组织。

---

## 一、项目功能介绍

### 1.1 项目概述

**逐光自习室管理系统**是一个基于 **Spring Boot + Vue 3** 的前后端分离 Web 应用，为自习室提供在线预约、座位管理、支付结算、会员与积分、学习统计等完整业务能力。

### 1.2 技术架构

| 层次 | 技术栈 |
|------|--------|
| 后端 | Spring Boot 2.7.14、MyBatis 2.3.1、MySQL 8.0、Maven、JWT、支付宝沙箱、Spring Scheduled |
| 前端 | Vue 3.3.4、Element Plus 2.4.2、Vue Router 4、Pinia 2、Axios、Vite 4.5.0 |
| AI 助手 | 通义千问（DashScope OpenAI 兼容接口） |

### 1.3 核心功能模块

#### 用户端

- **用户认证**：注册、登录（JWT）、个人信息与头像管理
- **自习室与座位**：查看自习室列表/详情、座位分布图、使用情况时间轴（普通用户不显示他人隐私信息）
- **预约**：选座预约、冲突检测、支付、取消/退款（30 分钟窗口）、入场凭证、积分兑换时长
- **长期租赁**：提交申请、支付、退款（2 小时窗口）
- **会员**：VIP/SVIP 购买与权益
- **积分**：每日签到、兑换预约、流水查询
- **留言板**：发帖、评论、点赞（支持匿名）
- **通知**：系统通知、未读提醒、自动刷新
- **学习统计**：个人时长统计、图表、排行榜
- **其他**：呼叫管理员、联系管理员、公告、使用记录、备忘录、额外收费、AI 助手（逐光 Agent）

#### 管理端

- **用户管理**：用户列表、搜索、启用/禁用、修改信息、修改日志
- **座位管理**：五区（A/B/C/D/E）座位状态、时间轴（含用户信息）
- **预约/租赁管理**：多条件筛选、审核长期租赁
- **价格配置**：预约价、租赁价、会员价、折扣、积分等动态配置
- **统计分析**：座位、预约、租赁、用户、收入统计
- **内容管理**：公告、留言板、通知推送
- **积分/呼叫/额外收费**等管理功能

## 三、部署步骤

### 3.1 MySQL 数据库部署

#### 3.1.1 数据库文件说明

项目提供两个 SQL 文件（位于 `db_sql/`）：

| 文件                                | 内容                    | 适用场景                                         |
| ----------------------------------- | ----------------------- | ------------------------------------------------ |
| `db_sql/studyroom_db_onlytable.sql` | **仅表结构**（20 张表） | **推荐**：干净部署                               |
| `db_sql/studyroom_db_data.sql`      | **仅业务数据**          | 需配合已有表结构使用；含测试用户、订单等演示数据 |

**推荐方案**：导入 **仅表结构** 文件，再手动插入管理员与基础业务数据（见下文）。

#### 3.1.2 创建并导入数据库，推荐方式B

**方式 A：命令行**

```bash
# 1. 登录 MySQL 并创建数据库
mysql -u root -p

CREATE DATABASE IF NOT EXISTS studyroom_db
  DEFAULT CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;
EXIT;

# 2. 导入表结构（在项目根目录 code/ 下执行）
mysql -u root -p studyroom_db < db_sql/studyroom_db_onlytable.sql
```

**方式 B：Navicat **

1. 新建数据库 `studyroom_db`，字符集 `utf8mb4`。
2. 运行 SQL 文件，选择 `db_sql/studyroom_db_onlytable.sql` 执行。

```bash
cd StudyroomSystem/backend
```

#### 3.2.2 配置 JDBC（MySQL 连接）

编辑 `backend/src/main/resources/application.yml` 中的数据源：

```yaml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/studyroom_db?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root          # 改为你的 MySQL 用户名
    password: your_password # 改为你的 MySQL 密码
```

**参数说明**：

| 配置项 | 说明 |
|--------|------|
| `localhost:3306` | MySQL 地址与端口，远程数据库改为实际 IP |
| `studyroom_db` | 数据库名，须与 3.1 节一致 |
| `serverTimezone=Asia/Shanghai` | 时区，避免时间偏差 |
| `allowPublicKeyRetrieval=true` | MySQL 8 本地开发常见需要 |

#### 3.2.3 配置支付宝沙箱（支付功能）

**官方入口**：

- 支付宝开放平台：https://open.alipay.com/
- 沙箱环境：https://open.alipay.com/develop/sandbox/app
- 开发文档：https://opendocs.alipay.com/
- 密钥生成工具说明：https://opendocs.alipay.com/common/02kkv7

**配置步骤概要**：

1. 注册并登录 [支付宝开放平台](https://open.alipay.com/)。
2. 进入 **控制台 → 开发助手 → 沙箱环境**，记录 **APPID**。
3. 使用官方工具生成 **RSA2** 密钥对，将**应用公钥**上传到沙箱应用的「接口加签方式」。
4. 在沙箱页面复制 **支付宝公钥**（不是应用公钥）。
5. 在 `application.yml` 中填写：

```yaml
alipay:
  app-id: 你的沙箱APPID
  app-private-key: |
    -----BEGIN PRIVATE KEY-----
    （你的应用私钥，PKCS8 格式，多行）
    -----END PRIVATE KEY-----
  alipay-public-key: |
    -----BEGIN PUBLIC KEY-----
    （沙箱环境中的支付宝公钥）
    -----END PUBLIC KEY-----
  sign-type: RSA2
  charset: UTF-8
  # 沙箱网关（与项目当前配置一致）
  gateway-url: https://openapi-sandbox.dl.alipaydev.com/gateway.do
  # 支付完成后浏览器跳回（后端同步回调）
  return-url: http://localhost:8080/api/reservation/pay/return
  # 支付宝服务器异步通知（本地 localhost 无法被支付宝访问，需内网穿透或部署到公网）
  notify-url: http://localhost:8080/api/reservation/pay/notify
  # 支付成功后跳回的前端地址
  frontend-base-url: http://localhost:3000
```

**说明**：

- 本地开发时 `notify-url` 使用 `localhost` 时，**异步回调通常收不到**，订单可能需依赖同步回调或手动查单；正式环境须改为 **HTTPS 公网地址**。
- 生产环境将 `gateway-url` 改为：`https://openapi.alipay.com/gateway.do`，并使用正式应用密钥。
- 更详细的图文说明见 [`docs/支付宝沙箱支付配置说明.md`](./支付宝沙箱支付配置说明.md)。

#### 3.2.4 配置通义千问 API Key（AI 助手）

**官方入口**：

- 阿里云百炼 / DashScope：https://dashscope.aliyun.com/
- 控制台（API Key）：https://dashscope.console.aliyun.com/
- 兼容 OpenAI 接口文档：https://help.aliyun.com/zh/model-studio/developer-reference/compatibility-of-openai-with-dashscope

**配置步骤概要**：

1. 注册阿里云账号，开通 **DashScope（灵积模型服务）**。
2. 在控制台创建 **API Key**。
3. 在 `application.yml` 中配置：

```yaml
ai:
  api-key: "sk-你的DashScope_API_Key"
  base-url: "https://dashscope.aliyuncs.com/compatible-mode/v1"
  model: "qwen3-max"          # 按控制台实际可用模型填写，如 qwen-plus、qwen-turbo 等
  timeout-seconds: 30
  enable-search: true         # 是否启用联网搜索（可选，可能产生额外费用）
  search-strategy: "agent"
```

**说明**：

- 后端通过 OpenAI 兼容的 Chat Completions 接口调用，配置类为 `AiModelConfig.java`。
- 若不使用 AI 助手，可暂时保留占位 Key，但调用 AI 接口时会失败；不影响预约、支付等核心功能。

## 五、常见问题

### 5.1 数据库连接失败

- 确认 MySQL 服务已启动，端口 3306 可访问。
- 核对 `application.yml` 中用户名、密码、库名 `studyroom_db`。
- MySQL 8 需使用 `com.mysql.cj.jdbc.Driver` 驱动（项目已配置）。

### 5.2 管理员无法登录

- 确认已执行 3.1.3 节 INSERT，且 `password` 为 MD5 值 `e10adc3949ba59abbe56e057f20f883e`（不是明文 `123456`）。
- 确认 `role = 1`、`status = 1`。

### 5.3 前端无法访问后端

- 确认后端已启动在 8080，且上下文路径为 `/api`。
- 开发环境依赖 Vite 代理；生产环境需 Nginx 将 `/api` 转发到后端。

### 5.4 支付失败或签名错误

- 检查 APPID、应用私钥、支付宝公钥是否来自同一沙箱应用。
- 私钥须为 PKCS8 格式，YAML 多行块保留完整 BEGIN/END。
- 参考 [`docs/支付宝沙箱支付配置说明.md`](./支付宝沙箱支付配置说明.md)。

### 5.5 AI 助手无响应

- 检查 `ai.api-key` 是否有效、账户是否开通 DashScope。
- 检查 `model` 名称是否与控制台可用模型一致。
- 查看后端日志中 AI 接口报错信息。

### 5.6 页面刷新 404（生产环境）

- Nginx 需配置 `try_files $uri $uri/ /index.html;` 以支持 Vue Router。

### 5.7 `npm install` 很慢或失败

- 使用国内镜像：`npm install --registry=https://registry.npmmirror.com`
- 确认 Node.js 版本 ≥ 16。
